SAÉ 3.01

Mettre en œuvre un système de transmission IoT

Introduction : Le suivi en temps réel dans des zones sans couverture GSM (4G/5G) nécessite des solutions spécifiques. Ce projet exploite la technologie LoRaWAN pour répondre aux besoins de suivi lors d'événements sportifs (marathons) grâce à des dispositifs basse consommation et longue portée.

🎯 Apprentissages Critiques :

  • Compétence RT2 : Connecter les entreprises et les usagers
  • • AC22.01 : Déployer et caractériser des systèmes de transmissions complexes

1. Description du projet

J'ai paramétré des capteurs IoT et configuré une passerelle (Gateway) LoRaWAN pour capter les trames. Ces données ont ensuite été acheminées vers un Network Server pour décoder la charge utile (payload) et permettre le suivi des participants sur une interface dédiée.

2. Présentation et justification des résultats

Le système est capable de remonter les informations des capteurs sur plusieurs kilomètres avec une consommation énergétique minimale.

Preuve 1 : Réception et décodage des trames (Payload) sur le Network Server.

3. Conclusion & Analyse Réflexive

📊 Auto-évaluation :

L'AC22.01 est validé. J'ai compris l'architecture complète d'un réseau IoT (Device -> Gateway -> Network Server -> Application).

🧠 Analyse Réflexive :

Expérience : Configurer les fréquences et le Spreading Factor (SF) a été complexe. J'ai réalisé que l'IoT impose un compromis strict entre la portée du signal et le débit des données.
