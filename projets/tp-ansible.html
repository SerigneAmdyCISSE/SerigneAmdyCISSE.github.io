Infrastructure as Code : Automatisation Ansible

Mise en œuvre d'une approche DevOps (Infrastructure as Code) pour remplacer la configuration manuelle (CLI) par des "Playbooks" automatisés. L'objectif : provisionner un routeur Cisco CSR1000v et un serveur web Debian Linux de manière reproductible.

Ansible YAML / Git Cisco IOS Debian Linux Netmiko (Python)

1. Contexte & Objectifs

Aujourd'hui, l'administration réseau moderne ne se fait plus équipement par équipement. Ce projet m'a permis de déployer un réseau hétérogène complet de manière 100% scriptée. Le but était d'utiliser Ansible pour lier le monde du réseau (Cisco) et celui des systèmes (Serveur Web Linux), le tout versionné sur un dépôt Git pour garantir la traçabilité des configurations.

2. Solutions Techniques Déployées

Le déploiement s'articule autour de playbooks structurés, s'appuyant sur un inventaire et des variables d'hôtes (Host_vars) :

  • Configuration du Routeur Cisco : Automatisation des interfaces réseaux, déploiement du service DHCP, et implémentation du NAT/PAT (dynamique et redirection de ports) via les modules cisco.ios.
  • Déploiement du Serveur Web (Linux) : Utilisation d'Ansible pour provisionner et démarrer les services d'un serveur Debian distant de manière transparente.
  • Idempotence des tâches : Logique de script garantissant que l'exécution multiple du playbook ne génère pas d'erreurs ou de configurations en doublon (ex: gestion intelligente de la création du pool DHCP).
  • Automatisation des tests (Netmiko) : Scripting d'une série de tests post-déploiement pour vérifier automatiquement la connectivité (Ping), la résolution de noms (DNS) et la réponse HTTP depuis la machine Ansible.

3. Résultats & Validation

L'exécution d'une simple commande ansible-playbook permet de reconstruire l'infrastructure réseau et applicative à l'identique en quelques secondes, sans aucune intervention manuelle en ligne de commande.

Scripting Réseau : Déploiement des adresses IP et de l'interface WAN en client DHCP via YAML.

Idempotence : Automatisation du serveur DHCP avec suppression préalable pour éviter les erreurs de doublons (Cisco IOS).

Logique avancée : Configuration dynamique du PAT (Port Forwarding) en itérant sur des listes avec l'attribut loop.

4. Bilan technique & Expertise acquise

Rigueur DevOps & Git : L'automatisation exige une rigueur syntaxique absolue. Coupler Ansible à un dépôt Git m'a permis d'intégrer la notion de "Version Control" au réseau de la même manière qu'un développeur avec son code source.

Le réseau dirigé par les tests (Test-Driven Networking) : L'intégration de la bibliothèque netmiko pour valider automatiquement la configuration du serveur (ping, requêtes DNS et web) post-déploiement a été une révélation. J'ai compris que provisionner un équipement ne suffit pas, il faut que le système valide lui-même que le service rendu aux utilisateurs est opérationnel.
